# Native

Any native code is written in Rust. There's two repositories:
[moq-rs](https://github.com/kixelated/moq-rs) and
[webtransport-rs](https://github.com/kixelated/webtransport-rs).

The project is split into a few crates:

|                                                                                         |                                                                                                                                                        |
| --------------------------------------------------------------------------------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
|                                                                            **moq-warp** | The media layer, used to publish and consume media streams.                                                                                            |
|                [**moq-transport**](https://docs.rs/moq-transport/latest/moq-transport/) | Implements the [MoQ Transport](https://datatracker.ietf.org/doc/draft-ietf-moq-transport/) draft and the underlying messages.                          |
| [**webtransport-quinn**](https://docs.rs/webtransport-quinn/latest/webtransport_quinn/) | Implements the [WebTransport draft](https://datatracker.ietf.org/doc/draft-ietf-webtrans-http3/) on top of [Quinn](https://github.com/quinn-rs/quinn). |
|                                                                **webtransport-generic** | A generic WebTransport trait, allowing moq-rs to use any QUIC library.                                                                                 |

We are currently working on ffmpeg and OBS integration. Check out the
[open issues](https://github.com/kixelated/moq-rs/issues) for a rough road map and potential
contributions!

# Web

The web code is written in Typescript. It's available on NPM as
[@kixelated/moq](https://www.npmjs.com/package/@kixelated/moq). There's currently no documentation
and the library is unstable; expect frequent changes.

It's split into folders depending on your use-case:

|                |                                                                                                                                                                                                                                                                                                                                                                                  |
| -------------: | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **contribute** | Captures media, encodes via [WebCodecs](https://developer.mozilla.org/en-US/docs/Web/API/WebCodecs_API), and transmits over [WebTransport](https://developer.mozilla.org/en-US/docs/Web/API/WebTransport).                                                                                                                                                                       |
|   **playback** | Receives media over [WebTransport](https://developer.mozilla.org/en-US/docs/Web/API/WebTransport), decodes via [WebCodecs](https://developer.mozilla.org/en-US/docs/Web/API/WebCodecs_API), and renders via [OffscreenCanvas](https://developer.mozilla.org/en-US/docs/Web/API/OffscreenCanvas) and [WebWorklet](https://developer.mozilla.org/en-US/docs/Web/API/AudioWorklet). |
|  **transport** | Implements [MoQ Transport](https://datatracker.ietf.org/doc/draft-ietf-moq-transport/) draft and the underlying messages.                                                                                                                                                                                                                                                        |

This project is using a lot of new web APIs. Chrome support is guaranteed but other browsers are
still catching up. See [issues](/issues) for more details.
